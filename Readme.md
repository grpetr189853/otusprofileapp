Оптимизированное приложение запускалось со флагами:
<br/>
-XX:+UnlockCommercialFeatures -XX:+FlightRecorder -XX:StartFlightRecording=duration=120s,filename=orig.jfr,settings=profile,dumponexit=true -XX:+UseCompressedOops
<br/>
Скриншоты и анализ на основе jmc5.
<br/>
Основные моменты:
<ul>
<li>В линуксе не удалось снять данные по записи в файл - они почему-то ен фиксировались. Более того, даже системные утилиты активности работы с диском не фиксировали</li>
<li>Не совсем корректно сравнивать приложения, т.к. в разное время может быть разный поток данных, более того ещё и разного размера. Единожды приложение отработалось за 40 секунд из-за плотности потока мелких пакетов</li>
<li>Много потоков - не всегда хорошо :)</li>
<li>Интересный опыт</li>
</ul>
